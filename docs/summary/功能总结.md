# LiteNote 功能总结

## 项目概述

LiteNote 是一个专注于"快速记录"与"知识关联"的轻量级全栈笔记系统，采用混合数据库架构，集成了 SQLite、LowDB、Keyv 和 Cassandra 四种数据存储引擎，实现了笔记管理、数据分析和用户行为追踪等核心功能。

## 核心功能模块

### 1. 笔记管理 (CRUD)

#### 1.1 创建笔记
- **功能描述**: 支持创建带有标题、内容（支持 Markdown）和标签的新笔记
- **存储策略**:
  - SQLite: 存储笔记元数据（ID、标题、标签、时间戳）
  - LowDB: 存储笔记正文内容（JSON 文件）
  - Keyv: 初始化阅读计数器
- **验证机制**: 标题、标签、内容格式验证
- **位置**: `src/app/actions/notes.ts:23-71`

#### 1.2 编辑笔记
- **功能描述**: 支持更新笔记的标题、内容和标签
- **存储策略**:
  - SQLite: 更新元数据字段
  - LowDB: 更新 JSON 文件内容
- **验证机制**: 字段存在性检查、格式验证
- **位置**: `src/app/actions/notes.ts:74-159`

#### 1.3 删除笔记
- **功能描述**: 物理删除笔记及其关联数据
- **存储策略**:
  - SQLite: 删除元数据记录
  - LowDB: 删除 JSON 文件
  - Keyv: 删除阅读计数器
- **位置**: `src/app/actions/notes.ts:162-194`

#### 1.4 笔记列表
- **功能描述**: 展示所有笔记，支持搜索和筛选
- **功能特性**:
  - 按更新时间倒序排列
  - 标题模糊搜索
  - 标签筛选
  - 分页支持
- **位置**: `src/app/actions/notes.ts:197-239`

#### 1.5 笔记详情
- **功能描述**: 查看笔记完整内容，支持 Markdown 渲染
- **功能特性**:
  - Markdown 内容渲染（支持 GFM）
  - 自动增加阅读计数
  - 异步记录用户行为日志
  - 显示创建/更新时间
  - 标签展示
- **位置**: `src/app/actions/notes.ts:242-293`

### 2. 内容交互功能

#### 2.1 阅读量统计
- **功能描述**: 实时统计笔记阅读次数
- **存储策略**:
  - Keyv: 使用 `note:{id}:views` 键存储计数
  - 原子递增操作
- **位置**: `src/app/actions/notes.ts:265`

#### 2.2 热榜展示
- **功能描述**: 展示阅读量最高的 Top 5 笔记
- **存储策略**:
  - Keyv: 扫描所有 `note:*:views` 键
  - SQLite: 获取笔记标题
- **展示位置**: 首页侧边栏
- **位置**: `src/app/actions/notes.ts:296-339`

#### 2.3 标签管理
- **功能描述**: 支持为笔记添加标签，按标签筛选
- **存储策略**: SQLite 存储逗号分隔的标签字符串
- **功能特性**:
  - 标签添加/编辑
  - 标签筛选
  - 标签展示（Badge 样式）
- **位置**: `src/app/actions/notes.ts:31-34, 100-105`

### 3. 数据分析功能

#### 3.1 行为埋点系统
- **功能描述**: 自动记录用户行为数据到 Cassandra
- **记录内容**:
  - 用户 ID
  - 时间戳
  - 笔记 ID
  - 行为类型（view, click, scroll, edit）
  - 停留时长
  - 设备类型（mobile, desktop, tablet）
  - 元数据（JSON 格式）
- **实现方式**:
  - 同步更新 Keyv 计数（快速响应）
  - 异步写入 Cassandra（不阻塞主流程）
- **位置**: `src/app/actions/analytics.ts:45-81`

#### 3.2 数据看板
- **功能描述**: 可视化展示用户行为数据
- **展示内容**:
  - 总记录数统计
  - 24 小时阅读量趋势
  - 平均每小时访问量
  - 设备分布（饼图）
  - 主要设备类型
  - 最近活跃记录列表
- **图表类型**:
  - 流量趋势图（柱状图）
  - 设备分布图（饼图）
- **位置**: `src/app/dashboard/page.tsx:1-572`

#### 3.3 数据浏览
- **功能描述**: 提供详细的行为日志数据表格
- **筛选功能**:
  - 按用户筛选
  - 按笔记筛选
  - 按行为类型筛选
  - 按设备类型筛选
  - 按时间范围筛选
- **分页支持**: 每页 50 条记录
- **位置**: `src/app/dashboard/data/page.tsx:1-427`

#### 3.4 用户轨迹追踪
- **功能描述**: 查看特定用户的所有操作历史
- **查询方式**: 按 user_id 查询，按时间倒序排列
- **展示内容**: 时间、行为类型、笔记、时长、设备
- **位置**: `src/app/actions/analytics.ts:120-165`

#### 3.5 设备分布统计
- **功能描述**: 统计不同设备类型的访问占比
- **统计维度**: mobile, desktop, tablet
- **输出格式**: 设备类型、数量、百分比
- **位置**: `src/app/actions/analytics.ts:170-216`

#### 3.6 数据聚合
- **功能描述**: 每日定时聚合统计数据
- **聚合内容**:
  - 总阅读量
  - 平均停留时长
  - 主要设备类型
  - 移动端/桌面端访问量
- **存储策略**: 写入 SQLite `daily_aggregated_stats` 表
- **位置**: `src/app/actions/analytics.ts:357-431`

### 4. 测试与维护功能

#### 4.1 测试数据生成
- **功能描述**: 生成模拟用户行为数据
- **生成策略**:
  - 随机用户 ID
  - 随机笔记 ID
  - 随机行为类型
  - 随机设备类型
  - 过去 24 小时内的时间戳
- **限制**: 界面生成最多 500 条，大量数据使用脚本
- **位置**: `src/app/actions/analytics.ts:281-348`

#### 4.2 数据清理
- **功能描述**: 清理测试数据，保留表结构
- **实现方式**: 使用 TRUNCATE 命令快速清空表
- **位置**: `src/app/actions/analytics.ts:473-491`

#### 4.3 Cassandra 重置
- **功能描述**: 完全重置 Cassandra（删除 Keyspace）
- **警告**: 危险操作，会删除所有数据
- **位置**: `src/app/actions/analytics.ts:497-514`

#### 4.4 健康检查
- **功能描述**: 检查 Cassandra 连接状态
- **检查内容**:
  - 连接可用性
  - Keyspace 是否存在
  - 表是否创建成功
  - 连接延迟
- **位置**: `src/app/actions/analytics.ts:440-468`

### 5. UI/UX 功能

#### 5.1 响应式布局
- **功能描述**: 适配不同屏幕尺寸
- **特性**:
  - 移动端隐藏侧边栏
  - 抽屉式菜单
  - 网格布局自适应

#### 5.2 Markdown 编辑器
- **功能描述**: 双栏编辑模式
- **特性**:
  - 左侧输入框（Markdown）
  - 右侧实时预览
  - 支持 GFM 语法

#### 5.3 搜索功能
- **功能描述**: 实时搜索笔记标题
- **实现**: 前端搜索框 + 后端 SQL LIKE 查询
- **位置**: `src/components/search-bar.tsx:1-1320`

#### 5.4 加载状态
- **功能描述**: 数据加载时显示占位符
- **场景**: 列表加载、详情加载、数据刷新
- **位置**: 各页面组件

#### 5.5 错误处理
- **功能描述**: 操作失败时的提示
- **方式**: Toast 提示、Alert 弹窗
- **覆盖范围**: 所有 CRUD 操作、数据查询

### 6. 定时任务

#### 6.1 每日聚合任务
- **功能描述**: 每日凌晨自动聚合前一日数据
- **实现**: Node.js Cron Job
- **位置**: `src/lib/cron.ts:1-100`

## 技术栈

### 前端
- **框架**: Next.js 14+ (App Router)
- **UI 组件**: Tailwind CSS + Shadcn/UI
- **Markdown**: react-markdown + remark-gfm
- **类型**: TypeScript

### 后端
- **运行时**: Node.js
- **数据库**:
  - SQLite (better-sqlite3)
  - LowDB (文档存储)
  - Keyv (键值缓存)
  - Cassandra (cassandra-driver)
- **定时任务**: node-cron

## 数据流

### 创建笔记流程
1. 用户填写表单（标题、内容、标签）
2. 前端调用 `createNote` Server Action
3. 验证输入数据
4. 生成 UUID 和时间戳
5. SQLite 插入元数据
6. LowDB 写入 JSON 文件
7. Keyv 初始化计数器
8. 返回成功，跳转至详情页

### 查看笔记流程
1. 用户访问笔记详情页
2. 后端调用 `getNoteDetail` Server Action
3. SQLite 查询元数据
4. LowDB 读取 JSON 文件
5. Keyv 递增阅读计数
6. 异步调用 `logBehavior` 记录到 Cassandra
7. 组装数据返回前端
8. 前端渲染 Markdown 内容

### 数据分析流程
1. 用户访问数据看板
2. 调用 `getDashboardStats` 获取统计数据
3. Cassandra 查询总记录数
4. Cassandra 查询 24 小时趋势
5. Cassandra 查询设备分布
6. 前端渲染图表

## 性能优化

1. **SQLite WAL 模式**: 提升并发性能
2. **Keyv 缓存**: 高频读写使用内存缓存
3. **异步日志记录**: 不阻塞主流程
4. **连接池管理**: Cassandra 连接复用
5. **批量插入**: 测试数据生成使用批量操作
6. **索引优化**: SQLite 创建必要索引

## 安全特性

1. **输入验证**: 所有用户输入进行格式验证
2. **SQL 注入防护**: 使用参数化查询
3. **XSS 防护**: Markdown 渲染自动转义
4. **错误处理**: 完善的 try-catch 机制
5. **连接重试**: Cassandra 连接失败自动重试

## 扩展性

1. **模块化设计**: 功能模块独立，易于扩展
2. **类型安全**: TypeScript 类型定义完整
3. **配置化**: 环境变量配置数据库连接
4. **插件化**: UI 组件可独立使用
5. **API 分层**: Server Actions 封装业务逻辑

## 总结

LiteNote 通过集成四种不同的数据存储引擎，实现了笔记管理、数据分析和用户行为追踪等完整功能。项目采用 Next.js App Router 架构，前后端一体化，提供了良好的用户体验和开发体验。混合数据库架构充分发挥了各存储引擎的优势，为不同类型的数据选择了最适合的存储方案。
